Replace the existing server-component scaffold (which uses `createClient` from server and has no interactivity) with a full 'use client' interactive page. Follow the Phase 2 CRUD pattern exactly from 03-RESEARCH.md (module-level `const supabase = createClient()`, useQuery v5 object form, useMutation with invalidateQueries).

**State:**
- `statusFilter: string` — default `''` (all), options: `'draft'|'sent'|'paid'|'overdue'|'cancelled'`
- `modal: boolean` + `editingInvoice: InvoiceRow | null` — single modal for create and edit
- `form` object: `{ contact_id: string, due_date: string, tax_rate: string, notes: string, items: Array<{description: string, quantity: string, unit_price: string}> }`

**Query:**
```typescript
useQuery({
  queryKey: ['invoices', statusFilter],
  queryFn: async () => {
    let q = supabase.from('invoices')
      .select('id, invoice_number, status, total, subtotal, tax_amount, issue_date, due_date, contact_id, notes, created_at')
      .is('deleted_at', null)
      .order('created_at', { ascending: false })
    if (statusFilter) q = q.eq('status', statusFilter)
    return q
  },
})
```

**Save mutation (create branch):**
- Compute `subtotal` = sum of `(qty * unit_price)` for all items
- Compute `tax_amount` = subtotal * (tax_rate / 100)
- Compute `total` = subtotal + tax_amount
- Generate `invoice_number` = `'FAC-' + Date.now().toString().slice(-6)`
- Insert into `invoices` with `owner_id` from `supabase.auth.getUser()`
- Then insert all `items` into `invoice_items` with the new `invoice_id`

**Save mutation (edit branch):**
- Update `invoices` row (subtotal, tax_amount, total, due_date, notes, tax_rate)
- Delete existing `invoice_items` where `invoice_id = editingInvoice.id`
- Re-insert updated items

**Soft delete mutation:** `supabase.from('invoices').update({ deleted_at: new Date().toISOString() }).eq('id', id)`

**Status filter tabs:** Render a row of pill buttons above the table (Tous, Brouillon, Envoyée, Payée, En retard, Annulée) that set `statusFilter` state. Active tab: `bg-violet-100 text-violet-700`, inactive: `text-gray-500 hover:text-gray-700`.

**Status badge colors** (copy from existing scaffold and keep the same STATUS_COLORS/STATUS_LABELS maps):
- draft: `bg-gray-100 text-gray-600`
- sent: `bg-blue-100 text-blue-700`
- paid: `bg-green-100 text-green-700`
- overdue: `bg-red-100 text-red-700`
- cancelled: `bg-orange-100 text-orange-700`

**Table columns:** N°, Statut, Montant (DZD formatted), Émise le, Échéance, Actions (Edit + Télécharger PDF link + Soft-delete)
PDF download link: `<a href={'/api/invoices/' + inv.id + '/pdf'} target="_blank">PDF</a>` — links to Route Handler created in Plan 02.

**Line items form:** Dynamic rows in the modal — "+" button adds a row, "×" removes it. Each row: description input, quantity number input, unit_price number input, computed total displayed. Running subtotal + tax + total shown below the items table.

**Modal size:** `size="xl"` (use Modal component from `@/components/ui/Modal`).

**PageShell props:** title="Factures", icon=FileText (from lucide-react), iconBg="bg-blue-100", iconColor="text-blue-600", actionLabel="Nouvelle facture", onAction={() => { setEditingInvoice(null); resetForm(); setModal(true) }}.

Use `PageShell`, `TableHead`, `EmptyRow`, `TableRow`, `Badge` from `@/components/ui/PageShell`.

**quotes/page.tsx — Replace server scaffold with 'use client' interactive page.**

Port `frontend/src/pages/facturation/Quotes.jsx` to Next.js with Supabase. Same Phase 2 CRUD pattern.

State: `modal: boolean`, `form: { contact_id: string, valid_until: string, tax_rate: string, subtotal: string, notes: string }`

Query:
```typescript
useQuery({
  queryKey: ['quotes'],
  queryFn: async () => supabase.from('quotes')
    .select('id, quote_number, status, total, valid_until, contact_id, converted_to_invoice_id, created_at')
    .is('deleted_at', null)
    .order('created_at', { ascending: false }),
})
```

**Create mutation:** Insert into `quotes` with:
- `quote_number`: `'DEV-' + Date.now().toString().slice(-6)`
- `total` = subtotal * (1 + tax_rate/100)
- `owner_id` from `supabase.auth.getUser()`

**Soft delete mutation:** `update({ deleted_at: new Date().toISOString() })`

**Convert to invoice mutation (convertMut):**
- Get user with `supabase.auth.getUser()`
- Insert new row into `invoices` using quote fields: `{ invoice_number: 'FAC-' + Date.now().toString().slice(-6), owner_id, contact_id: quote.contact_id, subtotal: quote.total, tax_rate: quote.tax_rate ?? 0, tax_amount: 0, total: quote.total, status: 'draft' }`
- Then: `supabase.from('quotes').update({ converted_to_invoice_id: newInvoice.id, status: 'accepted' }).eq('id', quote.id)`
- Invalidate both `['quotes']` and `['invoices']`
- Show toast: "Devis converti en facture"
