  <action>
Create `nextjs-app/supabase/migrations/002_seed_stages.sql`.

This migration inserts 6 default pipeline stages that are shared/global (not user-scoped) so they appear for all users. Because `pipeline_stages.owner_id` is NOT NULL, we need a strategy:

Option: Use a DO block that inserts stages only if none exist, using the first admin user's ID, OR use a sentinel approach. Since this is a seed migration for development, use a DO $$ block that selects the first user from auth.users and inserts stages for them — this is safe for dev environments. In production, stages per-user is the correct model.

SQL content:
```sql
-- Seed default pipeline stages
-- This runs once; stages are created for the first authenticated user found.
-- In production, the app auto-creates these on first deal page load if missing.
DO $$
DECLARE
  v_owner_id uuid;
BEGIN
  -- Get the first user from auth.users (the initial admin)
  SELECT id INTO v_owner_id FROM auth.users ORDER BY created_at ASC LIMIT 1;

  -- Only insert if this user has no stages yet
  IF v_owner_id IS NOT NULL AND NOT EXISTS (
    SELECT 1 FROM public.pipeline_stages WHERE owner_id = v_owner_id
  ) THEN
    INSERT INTO public.pipeline_stages (owner_id, name, position) VALUES
      (v_owner_id, 'Prospect',     0),
      (v_owner_id, 'Qualifié',     1),
      (v_owner_id, 'Proposition',  2),
      (v_owner_id, 'Négociation',  3),
      (v_owner_id, 'Gagné',        4),
      (v_owner_id, 'Perdu',        5);
  END IF;
END $$;
```

Also add a comment at the top explaining the purpose and that in the app, the deals page auto-seeds stages if the table is empty for the current user (handled in Plan 03).
