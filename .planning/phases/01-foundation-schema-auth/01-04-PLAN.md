  <how-to-verify>
    Test each flow in order. Use http://localhost:3000 for local tests.

    **1. Route protection**
    - Open incognito browser
    - Navigate to http://localhost:3000/
    - Expected: Redirected to http://localhost:3000/login
    - If NOT redirected: proxy.ts is broken

    **2. Email/password login**
    - On login page, enter a test account created in Supabase Dashboard → Authentication → Users
    - Submit
    - Expected: Redirected to http://localhost:3000/ (dashboard loads with sidebar)
    - Refresh page: session persists (not logged out)
    - If session doesn't persist: cookie is not being set (check @supabase/ssr setup)

    **3. Account lockout**
    - Logout, go back to login page
    - Enter correct email, wrong password 5 times
    - Expected: After 5 failures, next attempt shows "Account is temporarily locked" message
    - Verify in Supabase Dashboard → Table Editor → profiles: locked_until is set for that user
    - If lockout doesn't trigger: check record_failed_login RPC and check_login_allowed RPC

    **4. Google OAuth**
    - On login page, click "Continuer avec Google"
    - Expected: Redirected to Google's OAuth consent screen
    - Sign in with a Google account
    - Expected: Redirected back to http://localhost:3000/callback, then to http://localhost:3000/
    - If OAuth fails: check Supabase → Authentication → Providers → Google is enabled with correct Client ID/Secret
    - If redirect URI error: add http://localhost:3000/callback to Supabase → Authentication → URL Configuration → Redirect URLs

    **5. TOTP 2FA enrollment**
    - When logged in, navigate to http://localhost:3000/settings/2fa
    - Click "Activer la 2FA"
    - Expected: QR code appears (SVG rendered inline)
    - Open Google Authenticator or Authy on your phone, scan the QR code
    - Enter the 6-digit code
    - Expected: "2FA activée" confirmation shown

    **6. TOTP 2FA login verification**
    - Logout
    - Login with email/password
    - Expected: Redirected to http://localhost:3000/2fa (not dashboard) because TOTP is enrolled
    - Enter the 6-digit code from your authenticator app
    - Expected: Redirected to http://localhost:3000/ (dashboard)

    **7. Logout**
    - Click logout in the sidebar
    - Expected: Session cleared, redirected to /login
    - Try navigating back to /: should redirect back to /login

    **8. Vercel preview deployment**
    - Push current branch to GitHub (if not already done)
    - Vercel should auto-deploy (check https://vercel.com → your project → Deployments)
    - Open the preview URL
    - Repeat test #1 (route protection) on the preview URL
    - Expected: Same redirect behavior as local

    **Database verification (optional)**
    Run in Supabase Dashboard → SQL Editor:
    ```sql
    -- Verify RLS blocks anonymous access
    SET LOCAL ROLE anon;
    SELECT count(*) FROM public.contacts;
    -- Expected: 0 rows (or permission denied)

    -- Verify profiles trigger worked
    SELECT id, full_name, role FROM public.profiles LIMIT 5;
    -- Expected: rows for each user who signed up
    ```
