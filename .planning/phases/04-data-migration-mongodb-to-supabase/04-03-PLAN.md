# Plan 04-03: Validation, Runbook, and Cutover Documentation

## Goal

Write `migration/validate.py` to confirm record counts match between MongoDB and Supabase, zero FK violations exist, financial totals are consistent, and 10 random spot-checks per table pass. Write `migration/README.md` as the step-by-step runbook. Write `.env.production.example` with all required Supabase production keys.

## Context

Plans 04-01 and 04-02 produced the ETL pipeline and executed the data load. This plan closes the loop by making the migration verifiable and reversible. The `validate.py` script runs two data sources in parallel: it queries MongoDB for counts and Supabase for counts, then diffs them. FK integrity queries use `NOT EXISTS` subqueries against the Supabase PostgreSQL superuser connection. The `migration/README.md` is the authoritative runbook — it must be complete enough that a developer who has not read the research file can execute the entire migration from zero. The zero-downtime strategy (FR-07-5) and rollback plan (FR-07-6) are documented here: old Render deployment stays live, Next.js/Supabase runs in parallel, DNS/traffic cut-over only after validation passes.

## Tasks

### Task 1 — Write migration/validate.py

**File(s):**
- `migration/validate.py`

**Action:** Create

**Description:**

Write a validation script that: (1) counts records in every MongoDB collection and every Supabase table and reports deltas, (2) runs FK integrity NOT EXISTS queries (expecting 0 orphan rows), (3) checks invoice item totals vs invoice subtotals, (4) checks journal entry debit/credit balance, (5) verifies auth.users count equals profiles count, (6) runs 10 random spot-checks per table comparing a UUID-matched record, and (7) writes `migration/validation_report.json`.

```python
#!/usr/bin/env python3
"""
validate.py — Post-migration integrity checks.

Compares MongoDB counts vs Supabase counts for all 19 collections,
runs FK integrity queries, financial sanity checks, and spot-checks
10 random records per table.

Usage:
  cd migration/
  python validate.py

Output:
  migration/validation_report.json  — full results
  Exits with code 0 if all checks pass, 1 if any check fails.
"""

import os
import sys
import json
import random
import psycopg2
from pymongo import MongoClient
from dotenv import load_dotenv

load_dotenv(".env.migration")

# Collection name → Supabase public table name
COLLECTION_TABLE_MAP = {
    "users":           "profiles",          # auth.users handled separately
    "companies":       "companies",
    "contacts":        "contacts",
    "pipeline_stages": "pipeline_stages",
    "deals":           "deals",
    "tasks":           "tasks",
    "notes":           "notes",
    "invoices":        "invoices",
    "invoice_items":   "invoice_items",
    "quotes":          "quotes",
    "payments":        "payments",
    "accounts":        "accounts",
    "journal_entries": "journal_entries",
    "journal_lines":   "journal_lines",
    "departments":     "departments",
    "employees":       "employees",
    "leave_requests":  "leave_requests",
    "payslips":        "payslips",
    "google_tokens":   "google_tokens",
    "audit_logs":      "audit_logs",
}

FK_INTEGRITY_QUERIES = [
    ("companies orphaned owner_id",
     """SELECT COUNT(*) FROM public.companies c
        WHERE NOT EXISTS (SELECT 1 FROM auth.users u WHERE u.id = c.owner_id)"""),

    ("contacts orphaned owner_id",
     """SELECT COUNT(*) FROM public.contacts c
        WHERE NOT EXISTS (SELECT 1 FROM auth.users u WHERE u.id = c.owner_id)"""),

    ("contacts orphaned company_id",
     """SELECT COUNT(*) FROM public.contacts c
        WHERE c.company_id IS NOT NULL
          AND NOT EXISTS (SELECT 1 FROM public.companies co WHERE co.id = c.company_id)"""),

    ("deals orphaned stage_id",
     """SELECT COUNT(*) FROM public.deals d
        WHERE d.stage_id IS NOT NULL
          AND NOT EXISTS (SELECT 1 FROM public.pipeline_stages ps WHERE ps.id = d.stage_id)"""),

    ("invoice_items orphaned invoice_id",
     """SELECT COUNT(*) FROM public.invoice_items ii
        WHERE NOT EXISTS (SELECT 1 FROM public.invoices i WHERE i.id = ii.invoice_id)"""),

    ("journal_lines orphaned journal_entry_id",
     """SELECT COUNT(*) FROM public.journal_lines jl
        WHERE NOT EXISTS (SELECT 1 FROM public.journal_entries je WHERE je.id = jl.journal_entry_id)"""),

    ("journal_lines orphaned account_id",
     """SELECT COUNT(*) FROM public.journal_lines jl
        WHERE NOT EXISTS (SELECT 1 FROM public.accounts a WHERE a.id = jl.account_id)"""),

    ("employees orphaned department_id",
     """SELECT COUNT(*) FROM public.employees e
        WHERE e.department_id IS NOT NULL
          AND NOT EXISTS (SELECT 1 FROM public.departments d WHERE d.id = e.department_id)"""),

    ("leave_requests orphaned employee_id",
     """SELECT COUNT(*) FROM public.leave_requests lr
        WHERE NOT EXISTS (SELECT 1 FROM public.employees e WHERE e.id = lr.employee_id)"""),

    ("payslips orphaned employee_id",
     """SELECT COUNT(*) FROM public.payslips p
        WHERE NOT EXISTS (SELECT 1 FROM public.employees e WHERE e.id = p.employee_id)"""),
]

FINANCIAL_SANITY_QUERIES = [
    ("invoice items total mismatch",
     """SELECT COUNT(*) FROM (
          SELECT i.id
          FROM public.invoices i
          JOIN public.invoice_items ii ON ii.invoice_id = i.id
          GROUP BY i.id, i.subtotal
          HAVING ABS(i.subtotal - SUM(ii.total)) > 0.01
        ) AS mismatches"""),

    ("journal entry debit/credit imbalance",
     """SELECT COUNT(*) FROM (
          SELECT je.id
          FROM public.journal_entries je
          JOIN public.journal_lines jl ON jl.journal_entry_id = je.id
          GROUP BY je.id
          HAVING ABS(SUM(jl.debit) - SUM(jl.credit)) > 0.01
        ) AS imbalances"""),

    ("auth.users vs profiles count mismatch",
     """SELECT ABS(
          (SELECT COUNT(*) FROM auth.users) -
          (SELECT COUNT(*) FROM public.profiles)
        ) AS diff"""),
]


def check_counts(mongo_db, pg_conn) -> list:
    results = []
    for col, table in COLLECTION_TABLE_MAP.items():
        mongo_count = mongo_db[col].count_documents({})
        with pg_conn.cursor() as cur:
            cur.execute(f"SELECT COUNT(*) FROM public.{table}")
            pg_count = cur.fetchone()[0]
        match = (mongo_count == pg_count)
        result = {
            "check": f"count:{table}",
            "mongo": mongo_count,
            "supabase": pg_count,
            "delta": pg_count - mongo_count,
            "passed": match,
        }
        status = "PASS" if match else "FAIL"
        print(f"  [{status}] {table}: MongoDB={mongo_count}, Supabase={pg_count}, delta={result['delta']}")
        results.append(result)
    return results


def check_fk_integrity(pg_conn) -> list:
    results = []
    for name, query in FK_INTEGRITY_QUERIES:
        with pg_conn.cursor() as cur:
            cur.execute(query)
            count = cur.fetchone()[0]
        passed = (count == 0)
        status = "PASS" if passed else "FAIL"
        print(f"  [{status}] {name}: {count} orphan rows")
        results.append({"check": f"fk:{name}", "orphan_count": count, "passed": passed})
    return results


def check_financial_sanity(pg_conn) -> list:
    results = []
    for name, query in FINANCIAL_SANITY_QUERIES:
        with pg_conn.cursor() as cur:
            cur.execute(query)
            value = cur.fetchone()[0]
        passed = (int(value) == 0)
        status = "PASS" if passed else "FAIL"
        print(f"  [{status}] {name}: {value}")
        results.append({"check": f"financial:{name}", "value": int(value), "passed": passed})
    return results


def spot_check_table(mongo_db, pg_conn, col: str, table: str, id_map: dict, n: int = 10) -> list:
    """
    Pick n random MongoDB docs, look up their UUID in id_map, query Supabase,
    confirm the row exists. Does NOT compare field values (schema mismatch
    between Mongo and PG is expected for some fields).
    """
    docs = list(mongo_db[col].find({}))
    if not docs:
        return [{"check": f"spot:{table}", "passed": True, "note": "empty collection"}]

    sample = random.sample(docs, min(n, len(docs)))
    results = []
    for doc in sample:
        oid_str = str(doc["_id"])
        uuid = id_map.get(oid_str)
        if not uuid:
            results.append({"check": f"spot:{table}:{oid_str}", "passed": False,
                            "reason": "oid not in id_map"})
            continue

        pk_col = "user_id" if table == "google_tokens" else "id"
        with pg_conn.cursor() as cur:
            cur.execute(f"SELECT 1 FROM public.{table} WHERE {pk_col} = %s", (uuid,))
            found = cur.fetchone() is not None

        results.append({
            "check": f"spot:{table}:{oid_str}",
            "uuid": uuid,
            "passed": found,
        })
        if not found:
            print(f"    [MISS] {table} UUID {uuid} not found in Supabase")

    passed = sum(1 for r in results if r["passed"])
    print(f"  [INFO] {table} spot-check: {passed}/{len(results)} found")
    return results


def main():
    print("=== validate.py ===\n")

    # Load id_map
    with open("migration/id_map.json") as f:
        id_map = json.load(f)
    print(f"Loaded id_map: {len(id_map)} entries\n")

    # Connect
    mongo_client = MongoClient(os.environ["MONGO_URI"])
    mongo_db = mongo_client["erp_radiance"]
    pg_conn = psycopg2.connect(os.environ["SUPABASE_DB_DSN"])

    all_results = {}
    all_passed = True

    print("--- 1. Record count parity ---")
    count_results = check_counts(mongo_db, pg_conn)
    all_results["counts"] = count_results
    if any(not r["passed"] for r in count_results):
        all_passed = False

    print("\n--- 2. FK integrity (expect 0 orphan rows) ---")
    fk_results = check_fk_integrity(pg_conn)
    all_results["fk_integrity"] = fk_results
    if any(not r["passed"] for r in fk_results):
        all_passed = False

    print("\n--- 3. Financial sanity ---")
    financial_results = check_financial_sanity(pg_conn)
    all_results["financial_sanity"] = financial_results
    if any(not r["passed"] for r in financial_results):
        all_passed = False

    print("\n--- 4. Spot-checks (10 random records per table) ---")
    spot_results = {}
    for col, table in COLLECTION_TABLE_MAP.items():
        results = spot_check_table(mongo_db, pg_conn, col, table, id_map, n=10)
        spot_results[table] = results
        if any(not r["passed"] for r in results):
            all_passed = False
    all_results["spot_checks"] = spot_results

    mongo_client.close()
    pg_conn.close()

    # Write report
    report = {
        "passed": all_passed,
        "summary": {
            "counts_failed": sum(1 for r in count_results if not r["passed"]),
            "fk_violations": sum(r.get("orphan_count", 0) for r in fk_results),
            "financial_issues": sum(r.get("value", 0) for r in financial_results),
        },
        "details": all_results,
    }
    with open("migration/validation_report.json", "w") as f:
        json.dump(report, f, indent=2, default=str)

    print(f"\n=== Validation {'PASSED' if all_passed else 'FAILED'} ===")
    print(f"Report written to migration/validation_report.json")
    sys.exit(0 if all_passed else 1)


if __name__ == "__main__":
    main()
```

**Acceptance:**
- `migration/validate.py` exists and imports without error.
- Script contains `check_counts`, `check_fk_integrity`, `check_financial_sanity`, and `spot_check_table` functions.
- Script writes `migration/validation_report.json`.
- Script exits with code 1 if any check fails (so CI can detect failures).
- FK integrity queries cover all 10 critical FK relationships listed in the code.
- Financial sanity includes: invoice item total mismatch, journal entry debit/credit imbalance, auth.users vs profiles count.

# Radiance ERP — Migration Runbook

MongoDB Atlas → Supabase PostgreSQL

---

## Prerequisites

1. Python 3.11+ installed
2. MongoDB Atlas network access: `0.0.0.0/0` whitelist must be active on migration day
3. Supabase project with schema applied (`001_schema.sql` through `004_indexes.sql`)
4. The `postgres` superuser password from Supabase Dashboard → Settings → Database → Connection string (Direct)
5. Node 20+ and the Next.js app running on Vercel (the new stack must be deployed before cutover)

## Setup

```bash
cd migration/
pip install -r requirements.txt
cp .env.migration.example .env.migration
# Fill in all four values in .env.migration:
# MONGO_URI              = from Render env vars (MongoDB Atlas SRV string)
# SUPABASE_URL           = https://[PROJECT-REF].supabase.co
# SUPABASE_SERVICE_KEY   = from Supabase Dashboard → Settings → API → service_role key
# SUPABASE_DB_DSN        = from Supabase Dashboard → Settings → Database → Direct connection string
#                          Format: postgresql://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres
#                          If IPv6 fails, use Session Pooler connection string instead (same port 5432)
```

## Step 1 — Extract + Transform (generates SQL files)

```bash
python migration/migrate.py
```

Expected output: `migration/output/` directory with 21 `.sql` files named `02_profiles.sql` through `21_audit_logs.sql`.

Check: `ls migration/output/*.sql | wc -l` should print `21`.

## Step 2 — Test user import (one user first)

```bash
python migration/load_users.py --test-one
```

Expected output: `[1/1] OK user@example.com` and `user_id_map.json` with 1 entry.

**MANDATORY manual check:** Log in to the Next.js app at your Vercel URL using the test user's original email and password. If login succeeds, the bcrypt hash was saved correctly. Proceed to Step 3.

If login fails (password rejected): the hash-saving bug (GitHub issue #1678) is active on your GoTrue version. The script's SQL fallback will handle it — but verify by checking `migration/validation_report.json` after the full run.

## Step 3 — Import all users

```bash
python migration/load_users.py
```

Expected output: one `[N/total] OK email` line per user. `migration/user_id_map.json` updated.

## Step 4 — Load SQL data files

```bash
python migration/load_data.py
```

Expected output: `[OK] 02_profiles.sql` through `[OK] 21_audit_logs.sql`. Exits cleanly.

If a file fails, check the error message. Common causes:
- FK violation: a parent table was not loaded. Check that `load_users.py` completed (auth.users must exist before profiles).
- Unique constraint: duplicate data in MongoDB (most likely payslips). The `deduplicate_payslips()` function in migrate.py handles this — if it still fails, manually inspect the output SQL.

## Step 5 — Validate

```bash
python migration/validate.py
```

Expected output: all checks PASS. `migration/validation_report.json` written.

If count mismatches exist: check if any MongoDB collection was updated between Step 1 and now. Re-run `migrate.py` and `load_data.py` (scripts are idempotent via `ON CONFLICT DO NOTHING`).

## Step 6 — Cutover (DNS/Traffic switch)

Only proceed after Step 5 passes fully.

1. Update Vercel environment variables to production values (see `nextjs-app/.env.production.example`)
2. Trigger a Vercel redeploy
3. Smoke test on the production Vercel URL: login, view contacts, view invoices
4. Update DNS CNAME to point `app.yourdomain.com` → Vercel deployment URL (if applicable)
5. Announce to users that the new system is live

## Rollback Plan (FR-07-6)

The old Render deployment (`erpro-dz-api` + `erpro-dz-frontend`) stays live throughout this process. **Do not decommission Render until 2 weeks after cutover.**

If the Supabase migration has issues after cutover:
1. Revert DNS CNAME back to `erpro-dz-frontend.onrender.com`
2. Users are back on the old system immediately
3. MongoDB data is untouched (the migration scripts are read-only against MongoDB)
4. Investigate issues using `migration/validation_report.json`
5. Fix, re-run Steps 1-5, and retry cutover

**Zero data loss guarantee:** The migration scripts never write to MongoDB. MongoDB Atlas data is the source of truth until cutover is confirmed stable.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `psycopg2.OperationalError: could not connect` | IPv6 not available | Use Session Pooler DSN from Supabase Dashboard |
| `login with original password fails` | bcrypt hash not saved (issue #1678) | Check `validate.py` output; script uses SQL fallback automatically |
| `FK violation on load_data.py` | load_users.py did not complete | Re-run load_users.py, then re-run load_data.py |
| `unique constraint violated on payslips` | Duplicate payslips in MongoDB | Run: `python -c "from migration.transform import deduplicate_payslips; print('OK')"` to confirm deduplication is active |
| `count mismatch in validate.py` | MongoDB updated after migrate.py ran | Re-run migrate.py + load_data.py (idempotent) |
